# CivicEye AI India Security Policy

## Purpose
CivicEye AI India is committed to maintaining the security, privacy, and reliability of the project. This Security Policy explains which versions are currently supported, how to report vulnerabilities responsibly, and how security reports are handled.

## Supported Versions
The following versions are currently covered under this security policy:

| Version | Supported |
|---------|-----------|
| latest | ✅ Yes |
| earlier than 0.1 | ❌ No |

Only the latest maintained version is eligible for active security review and remediation.

## Reporting a Vulnerability
The CivicEye AI India team takes security issues seriously. To protect users and systems, vulnerabilities should not be reported through public issue trackers or public discussion threads.

### How to Report
Please report security vulnerabilities privately using the following contact channel:

**Email:** [security@civiceye.example.com](mailto:security@civiceye.example.com)

A good report should include:
- A clear description of the vulnerability.
- Steps required to reproduce the issue.
- The potential impact on users, systems, or data.
- Any suggested mitigation or fix, if available.

Providing complete and reproducible details helps the team investigate and resolve the issue more efficiently.

## Response Timeline
The project aims to handle security reports according to the following timeline:

| Stage | Target Time |
|-------|-------------|
| Acknowledgement | Within 48 hours |
| Status update | Within 7 days |
| Resolution | Within 30 days, where reasonably possible |

Actual resolution time may vary depending on the complexity and severity of the issue.

## Recognition
Responsible reporters may be credited in release notes or project acknowledgements unless anonymity is requested.


## Scope

The following security issues fall within the scope of assessment:

* Weaknesses in authentication and access control mechanisms.
* SQL injection and other database-related vulnerabilities.
* Cross-Site Scripting (XSS) attacks.
* Cross-Site Request Forgery (CSRF) vulnerabilities.
* Exposure of confidential or sensitive information.
* Misconfigured, vulnerable, or improperly secured API endpoints.

## Out of Scope

The following issues are generally excluded from the scope:

* Vulnerabilities affecting only third-party libraries or dependencies without proven impact on the application itself.
* Attacks that rely on social engineering techniques.
* Physical security threats or unauthorized physical access scenarios.


## Disclosure Policy
CivicEye AI India follows a responsible disclosure approach. Security researchers and reporters are requested to provide reasonable time for investigation, remediation, and deployment of a fix before making any public disclosure.

Public disclosure before a fix is available may put users, systems, or data at unnecessary risk and should be avoided.

## Commitment
CivicEye AI India appreciates responsible security reporting and values the efforts of individuals who help strengthen the safety of the project.
