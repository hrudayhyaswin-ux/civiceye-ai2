# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ Yes    |
| < 0.1   | ❌ No     |

## Reporting a Vulnerability

The CivicEye team takes security seriously. **Please do not open a public GitHub issue for vulnerabilities.**

### How to Report

Email: `security@civiceye.example.com` *(replace with your actual address)*

Include:
- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

### Response Timeline

| Stage           | Time       |
|-----------------|------------|
| Acknowledgement | 48 hours   |
| Status update   | 7 days     |
| Resolution      | 30 days    |

We will credit reporters in release notes (unless you prefer anonymity).

## Scope

**In scope:** Auth flaws, SQL injection, XSS, CSRF, sensitive data exposure, broken API endpoints.

**Out of scope:** Third-party dependency issues, social engineering, physical security.

## Disclosure Policy

We follow responsible disclosure. Please allow reasonable time for patching before public disclosure.

Thank you for helping keep CivicEye safe. 🙏
